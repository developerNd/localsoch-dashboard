‚ùå Issue Identified: Shop Details Button Stuck Loading

Problem: Shop details form button was stuck in loading state and not working properly.
Root Causes:
  • Vendor ID was undefined, causing vendor data query to not run
  • Form was disabled when no vendor existed
  • API request format was incorrect for FormData uploads
  • No automatic vendor creation for new sellers

‚úÖ Fixes Applied

1. Fixed API Request Format

Updated useUpdateVendor hook to handle both FormData and JSON data properly.

// Before: const strapiData = toStrapiFormat(data); const response = await apiRequest('PUT', `/api/vendors/${id}`, strapiData); // After: if (data instanceof FormData) { requestData = data; // Don't set Content-Type for FormData } else { const strapiData = toStrapiFormat(data); requestData = strapiData; headers = { 'Content-Type': 'application/json' }; }

2. Added Automatic Vendor Creation

When no vendor exists, automatically create one for the user.

// If no vendor exists, create one first if (!currentVendorId) { const vendorData = { name: data.name || 'My Shop', description: data.description || '', contact: data.contact || '', // ... other fields user: user.id, isActive: true, isApproved: false, status: 'pending' }; const newVendor = await createVendorMutation.mutateAsync(vendorData); currentVendorId = newVendor.data.id; }

3. Updated Button States

Removed dependency on vendorId for button enabling and updated loading states.

// Before: disabled={updateVendorMutation.isPending || !vendorId} // After: disabled={updateVendorMutation.isPending || createVendorMutation.isPending}

4. Added Query Invalidation

Added proper cache invalidation to refresh user data after vendor creation.

// Update the user context with the new vendor ID queryClient.invalidateQueries({ queryKey: ['/api/auth/me'] });
